The Tribal Education Network (T-E-N)

An Application of the Semantic Web in an e-Learning System

T-E-N is an educational delivery system for Pacific Northwest Tribes. It is an initiative between Tribal leaders and University of Washington Bothell,to help tribal students earn college credentials while in high school. The project is focused on providing a web interface application to help Native American Learners access to pre-college and college content. It involves creating an environment for course content development and deployment. The course content would be stored as a cloud based casebook repository or document system that has a student, teacher and family portal. Casebook would contain academic components such as subject area readings/materials, articles, video, audio, lecture etc. and cultural components that relate cultural stories, projects, culture, history to academic course concepts and objectives. Creating casebook would also involve developing learning games. Casebook artifacts will have to be linked to learning objects, topics, activities, projects etc. using semantic web technologies like RDF, SPARQL. Using semantic web technologies, TEN would link artifacts based on meaning rather than just the syntactic relation, encode meaning into artifacts which allows machine readability.

The goal of this project is to demonstrate the possibility of using semantic web as the infrastructure for e-Learning system, to make learning process more efficient and effective for use by tribal students. Semantic web is about making the web machine understandable, by explicitly the declaring the knowledge embedded in web resources. It is about building the appropriate infrastructure for intelligent agents to run around the web, performing complex searches based on context and meaning, for their users. Semantic web represents a promising technology for realizing eLearning requirements of just in time, at work place, on demand and customized process of e-learning. It is well suited for implementing requirements of Tribal Education Network an e-learning project, which requires linking cultural learning content from tribes to academic courses and learning objectives. Ontology based description of content, context and structure of learning materials benefits the providing and accessing of learning materials. The project demonstrates the use semantic web technologies, how they bring structure and meaning to learning materials, which happen to be well-suited for implementing semantic technology algorithms and solutions. This semantic search allows retrieval of more relevant documents when compared to keyword based search.

The process map for using learning materials provided by tribes is shown in the figure below. Learning material provided by tribes could be in any format, not necessarily a soft copy or one in digitized form. For e.g. a hardcopy of book, a photograph, a video cassette, audio cassette. In order to make it accessible through TEN, it needs to be digitized first. The digitized copy can then be uploaded as blob to TEN database. At the time of upload, digital rights management metadata will be added and stored in TEN semantic data warehouse (triple store). The upload process is done by intaker (TEN user role).

The second step, which is semantic markup/tagging of digitized content is done by annotator (TEN user role). At this step, annotator will link stories received from tribes with the artifacts/uploaded learning objects, through semantic markup. The annotations would be stored in triple store.

Figure 2. Process Map for TEN

Once the annotations are available, semantic search using SPARQL is performed to search and link the learning objects to academic courses, based on concepts annotated with the courses.

Learnings

The project helped me learn a new set of Semantic Web technologies RDF, SparQL, OWL. Designing architecture of an application integrated with semantic technologies was challenging but at the same time helped me apply many of the concepts I had learned from my courses in MSCSS. Some of the other learnings were - Designing and developing ontology for e-learning domain, in Protege tool, defining classes, data properties, object properties and relations; Setting up the development environment for using the semantic tools and technologies; Integrating semantic tools and technologies with a Java/J2EE project; Setting up Openlink Virtuoso and accessing it to insert, update, query and delete triples from java code using Jena Provider API; Implementing web based interfaces for creating and storing semantic annotation for learning material and querying those using SparQL queries; Learning a new query language SparQL and writing complex queries using FILTERS, OPTIONAL (joins).
